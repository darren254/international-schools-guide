ISG ARTICLE PRODUCTION PROMPT

For use when briefing AI to write or assist with any ISG article. The article's primary keyword, title tag, H1, meta description, URL slug, breadcrumbs, internal links, external links and schema are pre-defined in the ISG Content Plan. Refer to that document before writing.

===============================================================
SECTION 1 — ON-PAGE SEO RULES

KEYWORD PLACEMENT
[ ] Primary keyword in the first paragraph or within the first 100 words
[ ] Primary keyword in at least one H2
[ ] Primary keyword in the alt text of at least one image
[ ] Secondary / LSI keywords used naturally throughout — not forced
[ ] Only one H1 per page
[ ] H2s each target a secondary or related keyword — must read naturally
[ ] H3s used for sub-points within H2 sections — no keyword stuffing

CONTENT
[ ] Minimum 1,000 words for competitive keywords
[ ] 1,500–2,500 words for flagship pages
[ ] Answers the search intent in the first screen — don't bury the lede
[ ] No duplicate content from other pages on the site
[ ] Updated date visible if content is refreshed

IMAGES
[ ] At least one image per article
[ ] File name is descriptive e.g. best-british-schools-jakarta.jpg (not IMG_4521.jpg)
[ ] Alt text contains target keyword naturally
[ ] Images compressed (under 150kb where possible)

PAGE EXPERIENCE
[ ] Page loads under 3 seconds
[ ] Mobile-friendly layout
[ ] No intrusive popups blocking content on load

READABILITY
[ ] Flesch reading score appropriate for audience (aim for 60+)
[ ] No walls of text — use subheadings, bullet points, tables where they add clarity
[ ] Spell-checked and proofread

KEYWORD CANNIBALISATION CHECK
[ ] Confirm no other page on ISG already targets the same primary keyword
[ ] If overlap exists, consolidate or clearly differentiate the angle

INTERNAL LINKING ARCHITECTURE
[ ] Every article links back to its parent hub page (e.g. all Jakarta articles link to #1 Best International Schools in Jakarta)
[ ] Hub pages link explicitly to all relevant spoke articles — not just where the writer happened to remember
[ ] The hierarchy (hub → spoke) must be clear and consistent across every article
[ ] Cross-reference the Content Plan for required internal links TO and FROM before submitting
[ ] Internal links use descriptive anchor text — never "click here" or "this article"

STRUCTURED DATA / SCHEMA
[ ] Apply the schema type(s) specified in the Content Plan for each article
[ ] Common types: Article, BreadcrumbList, FAQPage, ItemList
[ ] Quiz components should be marked up where technically supported
[ ] Fee tables and comparison tables should use structured markup where possible
[ ] Flag schema requirements to the developer at the same time as submitting copy

FEES & CURRENCY
[ ] All fees expressed in USD throughout the article
[ ] IDR figures may be shown in parentheses where useful for local context
[ ] Exchange rate: IDR 16,826 = $1 USD (24 February 2026 — use this rate for all articles from this date forward)
[ ] State the exchange rate and conversion date clearly in the footer or methodology note
[ ] Fees must be sourced from official school fee schedules or verified third-party sources
[ ] BEFORE writing any fees article, check https://international-schools-guide.com for existing fee data already published on the site — use this as a starting point and supplement with direct school sources. Do not rely on memory or training data for fee figures.
[ ] Always include a "fees correct as of [month, year]" note

ACCURACY DISCLAIMER (required on every article)
Include the following at the bottom of every article, worded naturally:

  We work hard to make every figure, date and description on this page accurate.
  We don't always get it right. If you spot an error — a fee that's changed, a fact
  that's out of date, something we've got wrong — please tell us. Use the feedback
  button above or email us directly. We'll check it and update the article.

This note is genuine, not defensive. It builds trust. It is not an admission of unreliability.

SECTION 3 — WRITING RULES (HUMAN STYLE)

VOICE & TONE
[ ] Write like a smart, well-informed friend — not a textbook
[ ] Be direct. Say the thing. Don't warm up to it.
[ ] Have a point of view — don't sit on the fence to avoid controversy
[ ] Confident but not arrogant
[ ] Warm but not gushing
[ ] Authoritative but not academic
[ ] Never write to impress — write to be understood
[ ] Never condescend — don't explain things parents already know
[ ] Don't define things by what they are not. "ISJ is not trying to be JIS or BSJ" tells the reader nothing — say what it IS. Lead with the positive identity, not the negation. This applies to schools, areas, curriculums, everything. Also avoid the "This is not X — it is Y" construction anywhere in body text. AI writing massively over-indexes on this pattern. Just say what the thing is.
[ ] Article reference numbers (#1, #8, #12, #19, etc.) are for internal production use only. They must never appear as visible text in any published article. In markdown links, use them only as temporary href placeholders — e.g. [International School Fees in Jakarta](#12) — where the reader sees the article title, not the number.

FEES — EDITORIAL STANCE
[ ] Fee schedules at Jakarta's international schools are published openly — most schools post them on their websites or provide them on request
[ ] Do not imply or suggest that schools hide fees. This is not true. Framing like "the fees schools don't tell you about" perpetuates a false impression
[ ] The reason the full cost surprises families is that tuition is one of several separate line items — not because information is withheld
[ ] Explain the full cost clearly and factually. Do not frame it as a revelation
[ ] Use fees as a proxy for quality — and be direct about it. Higher fees mean higher teacher salaries, which means a stronger applicant pool and better-qualified staff. This correlation is real and significant. Never write "affordable doesn't mean low quality" or any equivalent — it is factually wrong and editorially cowardly. Lower fees mean lower teacher quality on average. Say so.
[ ] The corollary is also true: the higher the fee, the more families are paying for genuinely better-qualified, better-paid teachers — not just nicer buildings

CURRICULUM — CAMBRIDGE VS ENGLISH NATIONAL CURRICULUM
[ ] These are not two different things and should never be written as if they are
[ ] The English National Curriculum is the framework — what is taught
[ ] Cambridge (CAIE), Edexcel, Pearson and others are examination and delivery organisations — companies that have built content and assessments aligned to the English National Curriculum
[ ] When a school says it "follows Cambridge" it typically means it uses Cambridge's version of the content — not a different curriculum
[ ] IGCSE and A Levels are qualifications offered by these organisations, globally recognised and equivalent to standard English qualifications
[ ] Never write "Cambridge curriculum" and "English National Curriculum" as if they are opposites. They are the same curriculum delivered by different organisations
[ ] If a reader would be confused by this distinction, explain it briefly and move on

SENTENCES
[ ] Short sentences are almost always better than long ones
[ ] Vary sentence length — rhythm matters
[ ] One idea per sentence where possible
[ ] Never open with a question
[ ] Never open with a dictionary definition
[ ] Never open with "In recent years"
[ ] Never open with "When it comes to"
[ ] Never open with "Are you looking for"
[ ] Open with the most interesting thing you have to say
[ ] Or open with a specific fact, number or scene that earns attention
[ ] Get to the point within the first three sentences

FILLER — WATCH FOR THESE AND REMOVE WHERE POSSIBLE
[ ] "really" — almost always adds nothing
[ ] "very" — usually a signal to find a stronger word instead; fine occasionally
[ ] "quite" — vague and weak
[ ] "rather" — usually filler; fine when used for contrast
[ ] "just" — remove it and the sentence is almost always better
[ ] "actually" — HARD BAN. Never use it. Defensive, redundant, and a hallmark of AI-sounding copy.
[ ] "basically" — vague and condescending
[ ] "essentially" — usually filler
[ ] "literally" — almost always filler
[ ] "simply" — condescending
[ ] "of course" — condescending
[ ] "obviously" — condescending
[ ] "clearly" — often used to avoid explaining something
[ ] "needless to say" — then don't say it
[ ] "as you can see" — the reader can see
[ ] "in order to" — replace with "to"
[ ] "due to the fact that" — replace with "because"
[ ] "at this point in time" — replace with "now"
[ ] "in the event that" — replace with "if"

SCHOOL MARKETING LANGUAGE — NEVER USE UNLESS EXPLICITLY ASKED
These phrases appear in every school brochure. Using them makes the writing sound like it was generated by a school's own marketing department. Avoid all of them:
[ ] "world-class" / "world class"
[ ] "the whole child"
[ ] "in an ever-changing world" / "in a rapidly changing world"
[ ] "holistic education" / "holistic development"
[ ] "nurturing environment"
[ ] "lifelong learners"
[ ] "21st century skills"
[ ] "global citizens"
[ ] "passion for learning"
[ ] "empower students"
[ ] "vibrant community"
[ ] "state-of-the-art facilities"
[ ] "inclusive community"
[ ] "child-centred learning"
[ ] "innovative teaching"
[ ] "excellence in education"
[ ] "committed to excellence"
[ ] Any phrase a school could put on its own homepage without irony
[ ] "on a regular basis" — replace with "regularly"
[ ] "a large number of" — replace with "many"
[ ] "the majority of" — replace with "most"
[ ] "in close proximity to" — replace with "near"
[ ] "in spite of the fact that" — replace with "although"
[ ] "prior to" — replace with "before"
[ ] "subsequent to" — replace with "after"

BURSTINESS & RHYTHM
[ ] Don't write at the same pace for the whole article — vary it
[ ] After a long technical paragraph, use a short punchy one
[ ] After a dense section, give the reader a moment to breathe
[ ] Use a one-sentence paragraph deliberately — not by accident
[ ] Don't let three long paragraphs run together unbroken
[ ] Read the article aloud — if you run out of breath, the sentence is too long
[ ] If every sentence is roughly the same length, rewrite the section
[ ] A short sentence after a long one creates emphasis — use this intentionally
[ ] Vary how sections begin — not every H2 section should open the same way

PARAGRAPHS
[ ] 2–4 sentences max per paragraph
[ ] One idea per paragraph
[ ] Never stack three long paragraphs in a row — break it up
[ ] Use a one-sentence paragraph for emphasis when it earns it

SPECIFICITY
[ ] Specific always beats vague
[ ] Use real numbers, real names, real places
[ ] "$12,000 to $28,000" beats "fees vary widely"
[ ] "Three schools" beats "several schools"
[ ] "Founded in 1994" beats "established for decades"
[ ] If you can't be specific, say why — don't paper over it with vague language

STRUCTURE
[ ] Lead with the content parents actually want — schools, fees, decisions
[ ] Push generic context and background towards the bottom
[ ] Every section should earn its place — if it doesn't add something, cut it
[ ] Don't repeat the intro at the end — the conclusion should move things forward
[ ] If you're making a list, introduce it — don't just drop bullets with no context
[ ] Transition between sections naturally — don't use "Now let's look at..."
[ ] Don't summarise what you just said before moving to the next section

SCHOOL TIER HIERARCHY — ALWAYS APPLY
Jakarta's international schools sit in two clear tiers. Never conflate them.

TOP TIER: JIS, ISJ, BSJ — and probably AIS
These are the schools with the strongest reputations, best facilities, most qualified staff, and highest fees. When describing the premium end of the market, these are the reference points.

MID TIER: NAS, ACG, Nord Anglia (BISJ), NZ School Jakarta, and similar
Credible schools with their own strengths, but not in the same category. Never describe a mid-tier school as "flagship", "leading", or "top" — that framing is factually wrong and misleading to readers.

SOURCE CITATIONS — DIRECTORIES
[ ] Never cite competitor school directories (e.g. International Schools Database, Expatica school listings) as a source. These are ISG competitors. Link to official school websites or credible external sources (IBO, Cambridge, Kemdikbud) only.

HONESTY & EDITORIAL INTEGRITY
[ ] If something is unknown or contested — say so
[ ] If data is limited — say so
[ ] Don't overstate what the evidence shows
[ ] Don't write a glowing section on a school to balance a critical one — be fair, not balanced for its own sake
[ ] Don't hide a weak point inside a long paragraph hoping no one notices
[ ] Don't be patronising — don't explain things parents already know

FORMATTING AS WRITING
[ ] Use a subheading when you're genuinely changing subject — not to break up long blocks
[ ] Bold a word or phrase only if it truly needs emphasis — not for decoration
[ ] Tables are for data comparison — not for information that reads fine as prose
[ ] Bullet points are for genuine lists — not for paragraphs with the line breaks removed

HEADINGS
[ ] Never use headings that explain the obvious or patronise the reader — "Why Secondary Is Different", "Why This Matters", "What You Need to Know" all sound like AI wrote them and make the reader feel dumb
[ ] Headings should describe what the section contains, not lecture the reader about why they should care
[ ] Good: "Exit Qualifications Compared", "The Top Tier", "Settling In". Bad: "Why Early Years Matters More Than You Think", "Why This Decision Is Important"

WHAT AI DOES THAT HUMANS DON'T
[ ] Humans don't start every section by explaining what they're about to say
[ ] Humans don't end every section by summarising what they just said
[ ] Humans don't use three examples when one good one does the job
[ ] Humans don't hedge every opinion into meaninglessness
[ ] Humans don't write "This is a complex topic" — they just handle the complexity
[ ] Humans don't write "There are many factors to consider" — they consider them
[ ] Humans contradict themselves occasionally — and notice when they do
[ ] Humans get bored of their own voice and change it up
[ ] Humans leave some things unsaid
[ ] If the writing sounds like it was generated — rewrite it
